# Привет! Это мой проект для стажировки в QA Automation

Я создал этот репозиторий, чтобы на деле показать, как я учусь **автоматизации на Java**. Вместо скучной теории я взял задачу, которая реально важна для сервисов вроде **Yandex Pro** — проверку данных при регистрации.

### **Что именно я сделал:**
*   **Валидатор данных:** Написал логику в файле `UserValidator.java`, которая проверяет номер телефона (формат **+998** для Узбекистана), длину пароля и имени.
*   **Защита от вылетов:** Код не «падает», если приходят пустые данные или `null`. Для стабильного приложения это **критически важно**.
*   **Тест-кейсы:** В файле `Main.java` я прогнал разные ситуации — и когда всё правильно, и когда пользователь ошибся, чтобы убедиться, что программа всегда отвечает верно.

### **Почему это важно:**
Я понимаю, что работа **QA-инженера** — это не просто «искать баги», а делать так, чтобы сервис работал надежно. В этом небольшом коде я постарался показать свой **системный подход** и внимание к деталям.
